BlessedTexts API Documentation

Simple, reliable SMS sending API

Overview

The BlessedTexts API allows you to send SMS messages programmatically using HTTP requests. It provides a simple REST interface that can be integrated into web or mobile applications.

Base URL

https://api.blessedtexts.com/v1/

Authentication

Each request must include your unique API_KEY in the request header:

Authorization: Bearer YOUR_API_KEY

Endpoints

POST /send_sms/

Send a single SMS message to one or more recipients.

Parameters

  • phone_number – Recipient’s phone number (string)
  • message – Message text (string)

Example Request

POST /send_sms/
Content-Type: application/json
Authorization: Bearer YOUR_API_KEY

{
  "phone_number": "+254700000000",
  "message": "Hello from BlessedTexts!"
}

Example Response

{
  "status": "success",
  "message_id": "e5a2cbd2",
  "details": "Message sent successfully"
}
GET /message_status/<message_id>/

Retrieve the delivery status of a previously sent message.

Example Request

GET /message_status/e5a2cbd2/
Authorization: Bearer YOUR_API_KEY

Example Response

{
  "message_id": "e5a2cbd2",
  "status": "Delivered",
  "timestamp": "2025-10-06T13:45:00Z"
}

Error Responses

The API uses standard HTTP response codes to indicate success or failure:

  • 200 OK – Request successful
  • 400 Bad Request – Missing or invalid parameters
  • 401 Unauthorized – Invalid or missing API key
  • 500 Internal Server Error – Server-side issue
